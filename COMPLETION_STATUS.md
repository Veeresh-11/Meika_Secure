## 🎯 Sprint 2: WebAuthn Production (4-5 weeks)

### Database Schema (PRIORITY: BLOCKS EVERYTHING)

**File**: `migrations/003_webauthn_schema.sql`

```sql
-- WebAuthn Credentials (user's registered authenticators)
CREATE TABLE webauthn_credentials (
    id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL REFERENCES users(id),
    credential_id BYTEA NOT NULL UNIQUE,
    public_key BYTEA NOT NULL,
    counter BIGINT DEFAULT 0,
    transports TEXT[], -- USB, NFC, BLE, INTERNAL
    attestation_format TEXT, -- packed, fido-u2f, tpm, etc.
    aaguid TEXT, -- authenticator AAGUID
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_used_at TIMESTAMP,
    registered_device_id TEXT,
    UNIQUE(user_id, credential_id)
);

-- Registration Sessions (ephemeral, for challenge/response)
CREATE TABLE webauthn_registration_sessions (
    session_id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL REFERENCES users(id),
    challenge BYTEA NOT NULL,
    expires_at TIMESTAMP NOT NULL,
    state TEXT DEFAULT 'pending', -- pending, completed, expired
    device_id TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Authentication Sessions (ephemeral, for challenge/response)
CREATE TABLE webauthn_authentication_sessions (
    session_id TEXT PRIMARY KEY,
    user_id TEXT NOT NULL REFERENCES users(id),
    credential_id TEXT NOT NULL REFERENCES webauthn_credentials(id),
    challenge BYTEA NOT NULL,
    expires_at TIMESTAMP NOT NULL,
    state TEXT DEFAULT 'pending', -- pending, completed, expired
    device_id TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

-- Create indices
CREATE INDEX idx_webauthn_credentials_user_id ON webauthn_credentials(user_id);
CREATE INDEX idx_webauthn_credentials_credential_id ON webauthn_credentials(credential_id);
CREATE INDEX idx_webauthn_reg_sessions_user ON webauthn_registration_sessions(user_id);
CREATE INDEX idx_webauthn_auth_sessions_user ON webauthn_authentication_sessions(user_id);
```

## 🎯 Sprint 3: OAuth2/OIDC Provider (4-5 weeks) - PARALLEL TO SPRINT 2

### Endpoints to Implement:

**1. Authorization Endpoint** (6 hours)
- `POST /oauth/authorize`
- Request: client_id, scope, redirect_uri, state, code_challenge (PKCE)
- Response: Authorization code (6-min TTL) + state
- Database table: `oauth_authorization_codes`

**2. Token Endpoint** (8 hours)
- `POST /oauth/token`
- Request: code (authorization code), client_id, code_verifier (PKCE)
- Response: access_token (PQ-signed JWT) + id_token + refresh_token
- Issue tokens with DILITHIUM-3 signature

**3. Introspection Endpoint** (4 hours)
- `POST /oauth/introspect`
- Request: token + client credentials
- Response: token metadata (active, scope, exp, sub)

**4. JWKS Endpoint** (2 hours)
- `GET /oauth/.well-known/jwks`
- Response: Public keys in JWK format for each supported algorithm
- Include DILITHIUM-3, RS256, EdDSA keys

**5. OpenID Configuration** (2 hours)
- `GET /.well-known/openid-configuration`
- Response: OIDC server metadata (endpoints, algorithms, etc.)

### Database Schema:
```sql
CREATE TABLE oauth_clients (
    id TEXT PRIMARY KEY,
    name TEXT,
    client_secret BYTEA,
    redirect_uris TEXT[],
    allowed_scopes TEXT[],
    created_at TIMESTAMP
);

CREATE TABLE oauth_authorization_codes (
    code TEXT PRIMARY KEY,
    client_id TEXT REFERENCES oauth_clients(id),
    user_id TEXT REFERENCES users(id),
    scope TEXT,
    challenge_code TEXT, -- PKCE
    expires_at TIMESTAMP,
    used BOOLEAN DEFAULT FALSE
);

CREATE TABLE oauth_refresh_tokens (
    token TEXT PRIMARY KEY,
    user_id TEXT,
    client_id TEXT,
    scope TEXT,
    expires_at TIMESTAMP
);
```
